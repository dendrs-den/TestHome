# Contributing - InflightFlow

## Branching
- `main` - stable branch
- `develop` - integration branch
- `feature/<short-name>` - feature work
- `fix/<short-name>` - bug fixes

## Commit style
- `feat: ...`
- `fix: ...`
- `refactor: ...`
- `docs: ...`
- `test: ...`
- `chore: ...`

## Pull Request checklist
- Code compiles locally
- Lint and tests pass
- API contract changes documented
- Migration impact described
- Screenshots or short video attached for UI changes
- Rollback notes included for risky changes

## Definition of Done
- Feature works for expected user flow
- Edge cases handled and tested
- Logging for failure paths exists
- Documentation updated (`README` or `docs/*`)
- No TODOs left in merged code unless tracked in issue
